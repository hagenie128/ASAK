# 중앙 저장소 브랜치 정리 기록 (2026-07-19)

## 결론

`main`에서 별도 기능 개발이 진행된 것이 아니라, 문서 재구성·Figma 인수인계·워크로그·재료/메뉴 이미지 카탈로그가 하나의 브랜치에 함께 누적되어 큰 차이가 발생했다.

앱 실행 코드는 아래 저장소의 `main`을 기준으로 한다.

| 범위 | 기준 저장소 | 상태 |
| --- | --- | --- |
| 키오스크 UI | `ASAK-Kiosk` | `main` 반영 완료 |
| 관리자 UI | `ASAK-Admin` | `main` 반영 완료 |
| 백엔드 헬스체크 컬렉션 | `ASAK-back` | `main` 반영 완료 |
| 문서·에셋 허브 | `ASAK` | 별도 선별 반영 필요 |

## 병합하지 않은 이유

`handoff/0718-to-kiosk-continue`는 2026-07-19 기준 `origin/main`보다 12개 커밋 앞서고 8개 커밋 뒤처져 있다. 비교 범위에는 783개 파일(20,054 추가 / 13,365 삭제)이 포함된다.

대표적으로 아래 항목이 한 변경 범위에 섞여 있다.

- Notion 감사와 구현 가이드, Figma 화면·토큰 기록
- 팀 워크로그와 인수인계 문서
- 재료 아이콘·사진·메뉴 이미지 카탈로그
- 기존 문서의 대규모 이동·삭제
- `.prettierrc`, `.prettierignore`, 루트 README 같은 저장소 설정/안내 변경

따라서 제목이 문서 감사인 PR에 이미지와 저장소 재배치가 포함된 현재 상태는 검토·되돌리기·충돌 해결 비용이 너무 크다. 기존 `docs/notion-web-audit-v1` PR은 **Draft 유지 및 병합 금지**가 안전하다.

## 권장 분리 단위

| 우선순위 | 새 PR 범위 | 포함 기준 | 제외 기준 |
| --- | --- | --- | --- |
| P0 | Figma 인수인계 문서 | 화면 맵, 토큰, UI 구현 가이드 | 이미지 원본·프로젝트 설정 |
| P1 | 팀 작업 문서 | Worklog, 역할/작업 규칙, 인수인계 | 이전 문서 일괄 삭제 |
| P2 | 재료·메뉴 에셋 | 카탈로그, 라이선스/출처, 실제 사용 경로 | 문서 재배치와 혼합 커밋 |
| P3 | 오래된 문서 정리 | 삭제 후보 목록과 대체 링크를 먼저 제시 | 한 번에 대량 삭제 |

각 PR은 하나의 목적만 가져야 하며, `main` 최신 상태에서 새 브랜치를 만들어 변경 전후 링크와 파일 수를 확인한 뒤 병합한다.

## 다음 작업 규칙

1. 기능 저장소(`ASAK-Admin`, `ASAK-Kiosk`, `ASAK-back`)의 구현을 중앙 문서 PR에 다시 섞지 않는다.
2. 에셋은 코드에서 실제 참조할 위치와 라이선스/출처를 문서화한 뒤 별도 PR로 반영한다.
3. 삭제·이동은 대체 경로와 링크 검증 결과를 포함한 전용 PR에서만 수행한다.
4. 대규모 기존 PR은 새 PR들이 반영된 뒤, 남은 변경을 재비교하여 닫거나 범위를 축소한다.
