## TC 상세

### TC-001 먹고가기/포장 선택 및 orderType 유지 검증 (FWD-ORDER-001)

- **전제조건**: 키오스크 홈 화면 접근 가능
- **수행 절차**: 1) 홈(SCR-001)에서 매장(먹고가기) 선택 2) 메뉴 1건 장바구니 담기 3) 장바구니·주문확인(SCR-005)에서 orderType 확인 4) 컨펌 팝업 확인 후 API-005 body의 orderType 확인
- **기대 결과**: orderType(EAT_IN/TAKE_OUT)이 장바구니·주문 생성까지 유지
- **관련**: SC-014 · API-005 · SCR-001

### TC-002 화면 전환 5회 이내 주문 완료 흐름 검증 (FWD-UI-002)

- **전제조건**: 키오스크 정상 구동
- **수행 절차**: 메뉴선택→추천조합 기본값→장바구니·주문확인(컨펌 팝업)→결제(로딩), 화면 전환 5회 이내
- **기대 결과**: 5회 이내 화면 전환으로 주문 완료
- **관련**: SC-002 · API-002~006 · SCR-003~008

### TC-003 품절 판매 항목이 정상적으로 비활성화되는지 확인 (LMIS-MENU-002)

- **전제조건**: 특정 판매 항목이 품절 상태로 사전 설정됨
- **수행 절차**: 1) 관리자 품절 처리 2) 키오스크 메뉴/옵션 확인 3) SOLD OUT 표시 4) 선택 시도
- **기대 결과**: 품절 항목 회색+SOLD OUT 표시, 선택 불가
- **관련**: SC-003 · API-004, API-009 · SCR-003~004

### TC-004 결제 실패 시 원인 안내 및 장바구니 유지 확인 (FWD-PAY-002)

- **전제조건**: 결제 실패를 의도적으로 유발할 수 있는 테스트 모드
- **수행 절차**: 1) 장바구니 담기 2) 결제 3) 실패 응답 4) 안내 확인 5) 장바구니 유지
- **기대 결과**: 실패 원인 팝업/토스트 표시, 장바구니 유지
- **관련**: SC-005 · API-006 · SCR-007, SCR-012

### TC-005 스탬프 적립 확인이 결제 전후로 중복되지 않는지 검증 (LMIS-MEMBER-001)

- **전제조건**: 테스트용 멤버십 계정 존재
- **수행 절차**: 1) 멤버십 고객 결제 2) 적립 확인창 1회만 3) 결제 완료 후 적립 결과
- **기대 결과**: 결제 과정 중 1회만 확인, 적립결과 표시
- **관련**: SC-006 · API-006, API-018 · SCR-007~008

### TC-006 관리자 품절 처리가 관련 메뉴 전체에 즉시 반영되는지 검증 (LMIS-MENU-002)

- **전제조건**: 동일 판매 항목이 2개 이상 메뉴/옵션에 공통 존재
- **수행 절차**: 1) 관리자 품절 토글 ON 2) 키오스크 메뉴/옵션 목록 확인
- **기대 결과**: 품절 변경이 관련 메뉴/옵션 전체에 즉시 반영
- **관련**: SC-007 · API-009 · SCR-011, SCR-003~004

### TC-007 타임아웃 안내 후 자동 초기화 검증 (FWD-SYS-001)

- **전제조건**: 장바구니에 메뉴가 담겨 있고 일정 시간 입력 없음
- **수행 절차**: 1) 주문 중 대기 2) 타임아웃 안내 3) 홈 복귀 확인
- **기대 결과**: 타임아웃 후 홈 화면 자동 초기화, 장바구니 비움
- **관련**: SC-012 · — · SCR-013, SCR-005

### TC-008 접근성 설정 적용 검증 (FWD-UI-004)

- **전제조건**: 접근성 설정 화면 진입 가능
- **수행 절차**: 1) 글자크기/대비 변경 2) 메뉴/옵션/결제 화면 이동 3) 적용 확인
- **기대 결과**: 접근성 옵션이 전체 화면에 일관 적용
- **관련**: SC-013 · API-017 · SCR-014

### TC-009 관리자 로그인 검증 (LMIS-AUTH-001 (EXCLUDED))

- **전제조건**: 관리자 계정과 role_id 준비
- **수행 절차**: 1) 로그인 화면 2) 정상 계정 로그인 3) 잘못된 계정 시도
- **기대 결과**: 정상 계정은 관리자 화면, 오류 계정은 메시지 표시
- **관련**: SC-008 · — · SCR-015

### TC-010 관리자 메뉴 관리 목록/품절 상태 검증 (LMIS-MENU-004)

- **전제조건**: 메뉴/재료/옵션 샘플 데이터 등록
- **수행 절차**: 1) 메뉴 관리 목록 2) 품절 상태 확인 3) 상태 변경
- **기대 결과**: 목록·품절 상태 표시 및 키오스크 반영
- **관련**: SC-017 · API-009~011 · SCR-016

### TC-011 관리자 메뉴 등록/수정 검증 (LMIS-MENU-004)

- **전제조건**: 관리자 권한 로그인
- **수행 절차**: 1) 메뉴 등록/수정 2) 저장 3) 키오스크 메뉴 상세 확인
- **기대 결과**: 등록/수정 내용이 API-003/004 응답에 반영
- **관련**: SC-017 · API-012 · SCR-017

### TC-012 관리자 결제수단 설정 검증 (LMIS-PAY-001)

- **전제조건**: 결제수단 코드 등록
- **수행 절차**: 1) 노출/순서 변경 2) 저장 3) 고객 결제 화면 확인
- **기대 결과**: 활성 결제수단만 고객 화면에 표시
- **관련**: SC-017 · API-013~014 · SCR-018

### TC-013 관리자 매출 요약 조회 검증 (LMIS-ORDER-005)

- **전제조건**: 결제 완료 주문 데이터 존재
- **수행 절차**: 1) 매출 화면 2) 날짜 선택 3) 합계·주문수 확인
- **기대 결과**: 선택 날짜 기준 매출 합계·주문 수 표시
- **관련**: SC-018 · API-015 · SCR-019

### TC-014 관리자 주문 목록 조회 및 상태 변경 검증 (LMIS-ORDER-001)

- **전제조건**: 최소 1건 이상 주문 존재
- **수행 절차**: 1) 키오스크 주문 생성 2) 목록 최상단 확인 3) 상세 4) PREPARING 변경
- **기대 결과**: 목록·상세에서 상태 변경 즉시 반영
- **관련**: SC-008 · API-007~008 · SCR-009~010

### TC-015 영수증 출력 여부 선택 및 모의 프린터 요청 (RTOS-DEVICE-001)

- **전제조건**: 결제 승인 완료, orderId·orderNo 존재
- **수행 절차**: 1) 결제 완료 2) 영수증 출력 선택 3) API-019 4) 결과 확인
- **기대 결과**: 출력 선택 시 프린터 요청, 미선택 시 주문번호 표시
- **관련**: SC-015 · API-019 · SCR-008, SCR-020

### TC-016 포인트·쿠폰 적립 및 QR 할인 적용 (KSD-MEMBER-001 (EXCLUDED))

- **전제조건**: 멤버십 회원 또는 유효 쿠폰 QR 보유
- **수행 절차**: 1) 결제 진입 2) 적립 1회 확인 3) 쿠폰 스캔(선택) 4) 결제 5) 적립 결과
- **기대 결과**: 스탬프 1회 확인, 쿠폰 스캔 시 할인 반영
- **관련**: SC-016 · API-018, API-020 · SCR-007, SCR-021
