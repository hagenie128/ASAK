# 목적

이 페이지는 키오스크 프로젝트의 화면 설계를 데이터베이스로 관리합니다.

실제 화면 디자인과 클릭 가능한 프로토타입은 Figma에서 제작하고, Notion에서는 화면 목록, 설계 상태, 담당자, 관련 요구사항, 관련 시나리오, 관련 API, 테스트 연결 정보를 관리합니다.

## 관리 방식

- 화면 목록 DB: 화면 단위로 상태와 연결 정 no보를 관리합니다.
- Figma 링크: 화면별 프레임 링크와 전체 프로토타입 링크를 등록합니다.
- 화면은 FWD 단계에서는 프로토타입 중심, KSD 단계에서는 API 연동 기준으로 관리합니다.

## 9주 프로젝트 기준 화면 흐름

Week 5 MVP에서는 고객 키오스크 주문 흐름과 관리자 주문 확인 흐름을 우선 구현하고, 후반 주차에는 품절 관리, 세트/태그/영양/알레르기 표시, 장치/RTOS 확장 여부를 단계적으로 검토합니다.

## 회의 확정 화면 흐름

### 고객용 키오스크

```
홈 (매장·포장) [SCR-001, SCR-002 병합]
→ 메뉴 선택 (SCR-003)
→ 메뉴 상세 / 옵션 선택 (SCR-004)
→ 장바구니·주문확인 (SCR-005, SCR-006 병합)
→ 결제 (SCR-007)
→ 주문 완료 (SCR-008)
```

### 관리자 화면

```
관리자 주문 관리
→ 관리자 주문 상세
→ 주문 상태 변경
```

### 옵션기능

```
관리자 판매 항목 품절 관리
→ 품절 상태 변경
→ 키오스크 화면 반영
```

00. Cover

## 화면 상태값

기획중 / 와이어프레임 / 디자인중 / 프로토타입완료 / 개발중 / 개발완료 / 수정필요 / 제외 / **병합됨**

## 화면 구분

키오스크 / 관리자 / 옵션기능 / 공통 / 장치 테스트 / 오류예외

## 작성 기준

1. 화면 ID를 부여합니다.
2. 화면 설명과 입출력 데이터를 작성합니다.
3. 관련 요구사항, 시나리오, API, 테스트 ID를 연결합니다.
4. Figma 프레임 링크와 Prototype 링크를 등록합니다.
5. 개발이 끝나면 상태를 개발완료로 변경합니다.
6. 요구사항/시나리오가 있는데 화면이 필요한 경우에는 화면 설계 DB에 반드시 추가합니다.
