## 1. 왜 Integration 토큰이 필요한가?

Notion **게스트**는 [my-integrations](https://www.notion.so/my-integrations)에서 **자신만의 Integration을 만들 수 없습니다.** API로 워크로그 DB에 행을 추가·수정하려면 **워크스페이스 소유자 또는 관리자(팀 리드)** 가 만든 **Internal Integration** 토큰이 필요합니다.

| 역할 | 할 일 |
| --- | --- |
| **팀 리드** (Owner/Admin) | Integration 생성 → 토큰 1회 안전 공유 → DB에 Integration 연결 |
| **팀원** (게스트·Free 포함) | 받은 토큰을 **로컬 환경 변수**에만 설정 → 스크립트 실행 |

> 토큰은 **비밀번호와 같습니다.** Git·Slack 공개 채널·스크린샷에 올리지 마세요.
>

## 2. Free 플랜 vs 게스트

| 구분 | Free 플랜 (본인 워크스페이스) | 게스트 (ASAK 공유 워크스페이스) |
| --- | --- | --- |
| Notion UI에서 페이지·DB 편집 | ✅ (초대된 페이지 범위) | ✅ |
| Integration **직접** 생성 | ✅ (본인 워크스페이스만) | ❌ |
| 팀 리드 Integration 토큰 사용 | ✅ (연결된 DB에 한함) | ✅ |
| API 동기화 스크립트 | 토큰 있으면 가능 | 토큰 있으면 가능 |

**정리:** Free 플랜이어도 **게스트라서** Integration을 못 만드는 것이지, Free라서 API를 못 쓰는 것은 아닙니다. 팀 리드가 만든 **공유 Internal Integration 토큰**을 쓰면 REST API 동기화가 됩니다.

## 3. 팀 리드 — Integration 설정 (1회)

### 3-1. Integration 생성

1. [my-integrations](https://www.notion.so/my-integrations) 접속 (팀 리드 계정)
2. **+ New integration** → 이름 예: `ASAK Worklog Sync`
3. **Type:** Internal
4. **Associated workspace:** ASAK 키오스크 프로젝트 워크스페이스 선택
5. **Capabilities:** `Read content`, `Update content`, `Insert content` (최소 Insert + Update)
6. **Submit** 후 **Internal Integration Secret** (`secret_…`) 복사

### 3-2. DB에 Integration 연결 (필수)

API는 **연결(Connect)된 페이지/DB**만 접근할 수 있습니다.

1. [📅 일일 워크로그 DB](../../%F0%9F%93%85%20%EC%9D%BC%EC%9D%BC%20%EC%9B%8C%ED%81%AC%EB%A1%9C%EA%B7%B8%20eeae4beb07ad4051928a87de0ea4c8f9.md) 열기
2. 우측 상단 **`···`** → **연결** (Connections) → 방금 만든 Integration 추가
3. (선택) 상위 [ASAK 키오스크 프로젝트](../../../ASAK%20%ED%82%A4%EC%98%A4%EC%8A%A4%ED%81%AC%20%ED%94%84%EB%A1%9C%EC%A0%9D%ED%8A%B8%20cd951ef04f0b82b081de019cc9a4c580.md) 페이지에도 연결

### 3-3. 토큰 안전 공유 (1회)

1. **1Password / Bitwarden** 등 팀 비밀 저장소에 `NOTION_TOKEN` 항목 생성
2. 팀원에게 **초대 링크** 또는 **대면 전달** (카톡·공개 이슈에 붙여넣기 금지)
3. 유출 시 Integration 페이지에서 **Secret 재발급(Rotate)** 후 팀에 재공유

## 4. 팀원 — 환경 변수 설정

### Windows (PowerShell — 현재 세션만)

```powershell
cd C:\ha-team
$env:NOTION_TOKEN = "secret_xxxxxxxx"
```

### Windows (영구 — 사용자 환경 변수)

1. **시작** → "환경 변수" 검색 → **시스템 환경 변수 편집**
2. **사용자 변수** → **새로 만들기** — 이름: `NOTION_TOKEN`, 값: `secret_xxxxxxxx`
3. 터미널·Cursor **재시작**

### macOS / Linux

```bash
# ~/.zshrc 또는 ~/.bashrc
export NOTION_TOKEN="secret_xxxxxxxx"
```

적용: `source ~/.zshrc`

### .env 파일 (선택, Git 제외)

저장소 루트 `.env` (커밋 금지): `NOTION_TOKEN=secret_xxxxxxxx`

```powershell
cd C:\ha-team
python asak-data/scripts/verify_notion_token.py
```

## 6. 워크로그 작성 방법 (팀원)

ASAK는 **Git 정본 + Notion 캘린더** 하이브리드입니다.

### 방법 A — 권장 (Git daily → 자동 sync)

```powershell
cd C:\ha-team
python worklog/scripts/init_daily.py
# worklog/daily/{본인}/YYYY-MM-DD.md 편집
python worklog/scripts/sync_daily_to_notion.py --date today
# 또는 Windows 래퍼
.\worklog\scripts\sync_today.ps1
```

- 같은 **날짜 + 담당** 행이 있으면 **갱신**, 없으면 **생성**
- Notion [캘린더 뷰](../../%F0%9F%93%85%20%EC%9D%BC%EC%9D%BC%20%EC%9B%8C%ED%81%AC%EB%A1%9C%EA%B7%B8%20eeae4beb07ad4051928a87de0ea4c8f9.md)에서 확인

### 방법 B — CLI로 Notion에 직접 1건 추가

```powershell
python asak-data/scripts/create_worklog.py `
  --date today `
  --author 이하진 `
  --task "SCR-003 메뉴 옵션 UI 구현" `
  --ref "SCR-003, WBS-015" `
  --hours 3 `
  --notes "옵션 그룹 검증 로직 추가"
```

### 방법 C — 토큰 없음 (Cursor Notion MCP)

1. `python worklog/scripts/sync_daily_to_notion.py --date today --json` 으로 payload 생성
2. Cursor에서 Notion MCP로 upsert → [`guide-mcp-sync.md`](https://github.com/hagenie128/ASAK/blob/main/worklog/guide-mcp-sync.md)

### 방법 D — Notion UI만 (Free·게스트 친화)

1. [📅 일일 워크로그 DB](../../%F0%9F%93%85%20%EC%9D%BC%EC%9D%BC%20%EC%9B%8C%ED%81%AC%EB%A1%9C%EA%B7%B8%20eeae4beb07ad4051928a87de0ea4c8f9.md) 열기
2. **새 페이지** 또는 **`_template`** 템플릿 복제
3. 필드 수동 입력: **날짜**, **담당**, **요약**, **WBS**, **블로커**
4. 상세 내용은 Git `worklog/entries/{이름}/` 에 12섹션으로 작성 (선택)

## 7. DB 스키마 (📅 일일 워크로그)

| Notion 속성 | 타입 | CLI / 스크립트 매핑 |
| --- | --- | --- |
| 제목 | title | `{날짜} {담당} 일일` (자동) |
| 날짜 | date | `--date` |
| 담당 | select | `--author` (이하진, 김나연, 박유진, 강민준, 미지정) |
| 요약 | text | `--task`  • `--hours`  • `--notes` |
| WBS | text | `--ref` (SCR-xxx, SC-xxx, WBS-xxx) |
| Git daily | url | daily sync 시 자동 / `--git-daily` |
| 블로커 | checkbox | `--blocker` |
- **database_id:** `eeae4beb07ad4051928a87de0ea4c8f9`
- **설정 파일:** [`worklog/notion_config.json`](https://github.com/hagenie128/ASAK/blob/main/worklog/notion_config.json)

> **투입 시간(hours)** 은 DB 전용 컬럼이 없어 **요약** 필드에 `(3h)` 형태로 붙입니다.

## 8. DevCopilot과의 관계

DevCopilot에는 **별도 워크로그 모듈이 없습니다.** WBS·화면·시나리오는 DevCopilot/Notion 허브에서 관리하고, **일일 작업 기록**은 Git `worklog/` + Notion **📅 일일 워크로그** 로 관리합니다.

## 9. 보안 체크리스트

- [ ]  `NOTION_TOKEN`을 `.env`, 환경 변수, 1Password에만 보관
- [ ]  `.env` · `secret_` 문자열을 **Git에 커밋하지 않음**
- [ ]  토큰 유출 시 즉시 [Integrations](https://www.notion.so/my-integrations)에서 **재발급**
- [ ]  퇴사·프로젝트 종료 시 Integration 연결 해제 검토

## 10. 문제 해결

| 증상 | 원인 | 해결 |
| --- | --- | --- |
| `NOTION_TOKEN not set` | 환경 변수 미설정 | §4 참고 후 터미널 재시작 |
| `401 unauthorized` | 잘못된 토큰 | 팀 리드에게 최신 secret 요청 |
| `404 object_not_found` | DB에 Integration 미연결 | §3-2 연결 작업 |
| `validation_error` (담당) | select 옵션에 없는 이름 | 이하진·김나연·박유진·강민준·미지정 중 선택 |
| Business plan upsell (MCP) | MCP 쿼리 제한 | REST API (`NOTION_TOKEN`) 경로 사용 |
