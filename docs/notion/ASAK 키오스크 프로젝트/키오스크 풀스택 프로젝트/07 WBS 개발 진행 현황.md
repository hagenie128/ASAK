# 07. WBS / 개발 진행 현황

# 목적

이 페이지는 ASAK 키오스크 프로젝트의 **9주** 개발 진행 현황을 WBS 데이터베이스로 관리합니다.

WBS는 프로젝트를 작은 작업 단위로 나누고, 각 작업의 담당자, 기간, 상태, 산출물, 완료 기준을 추적하기 위한 표입니다.

> 전체 프로젝트는 **9주**(7/2~9/2)입니다. **Week 1~5**는 키오스크 MVP 체크포인트(Week 5: SCR-001~008, 8/1 목표)이며, 이후 주차에는 관리자 기능, 옵션기능, 통합 테스트, 장치/RTOS 확장, **9/2 최종 발표**를 단계적으로 진행합니다.
>

# WBS 컬럼 의미

| 컬럼 | 의미 | 작성 예시 |
| --- | --- | --- |
| 단계 | 이 작업이 속한 프로젝트 단계/산출물 흐름입니다. “언제/어느 흐름에서 필요한 작업인가”를 봅니다. | FWD, LMIS, KSD, TEST |
| 구분 | 실제 작업 성격입니다. “무슨 종류의 일을 하는가”를 봅니다. | 프론트엔드, 백엔드, DB, 문서, 테스트 |
| 비고 | 완료 기준, 주의사항, 연결 문서/API/화면을 적습니다. 비워두지 않는 것을 원칙으로 합니다. | API-004 응답과 옵션 선택 화면이 동일한 옵션그룹 순서로 표시되면 완료 |

# 비고 작성 규칙

- 비고에는 “작업 설명”보다 “완료 기준”을 우선 적습니다.
- 관련 문서, 요구사항 ID, API ID, 화면 ID, DB 테이블명을 함께 적습니다.
- 비고 예시: `API-005 주문 생성 시 orders/order_item/order_item_option/item_exclusion 저장 확인까지 완료`.
- 비고 예시: `품절 기준: CORE/BASE는 메뉴 주문불가, DEFAULT는 품절 뱃지, 옵션항목은 해당 옵션 선택불가`.

# 관리 기준

- 작업은 가능한 작게 나눕니다.
- 각 작업에는 담당자를 지정합니다.
- 시작일과 종료일을 입력합니다.
- 상태와 진척률은 데일리 체크인 때 업데이트합니다.
- 완료된 작업은 관련 산출물과 완료 기준을 비고에 남깁니다.
- 기본기능 요구사항은 관련 시나리오, 화면, API 또는 WBS 중 최소 1개 이상 연결합니다.
- API 없이 프론트 상태만으로 가능한 작업도 WBS에는 별도 작업으로 등록합니다.
