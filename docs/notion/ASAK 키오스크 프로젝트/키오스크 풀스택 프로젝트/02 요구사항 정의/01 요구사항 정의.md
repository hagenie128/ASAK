# 목적

이 페이지는 키오스크 프로젝트의 요구사항을 데이터베이스로 관리합니다.

요구사항은 문장으로만 정리하면 구현 상태, 우선순위, 관련 화면/API/DB와 연결해서 보기 어렵습니다. 따라서 요구사항 목록은 인라인 DB로 관리합니다.

## 관리 기준

- 요구사항 ID를 부여합니다.
- 기능, 비기능, 화면, DB, API, 장치, 관리자 요구사항으로 구분합니다.
- 우선순위를 상/중/하로 정합니다.
- 관련 화면, API, DB 테이블을 함께 적습니다.
- 구현 상태를 계속 업데이트합니다.
- 최종 테스트 시 요구사항이 구현되었는지 확인합니다.

## 상태값

예정 / 진행중 / 검토중 / 구현완료 / 테스트완료 / 제외

## 요구사항 작성 예시

고객은 키오스크에서 메뉴 또는 예약 대상을 선택할 수 있어야 한다.

관리자는 주문/예약/결제 내역을 조회할 수 있어야 한다.

Spring Boot는 영수증 출력 요청 시 DB에서 주문/결제 내역을 조회한 뒤 모의 프린터 또는 RTOS Simulator로 출력 데이터를 전달해야 한다.
