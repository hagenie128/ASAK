# 관리자 판매 항목 품절 관리

Figma 링크: https://www.figma.com/design/iqaoVwFjFE6Zq1WpOVgjeG/kiosk_design?node-id=75-19
개발 완료: No
관련 API: API-002 GET /api/menus, API-003 GET /api/menus/{menuId}, API-004 GET /api/menus/{menuId}/options, API-009 PATCH /api/admin/sold-out-items
관련 시나리오: 판매 항목 품절 관리 흐름(옵션기능)
관련 요구사항: LMIS-MENU-001, LMIS-MENU-002
관련 테스트: TC-006
구분: 관리자
단계: KSD
도메인: 식당
비고: 옵션기능 화면. 메뉴 상세/API-004 옵션 응답에서 ingredientId 확인 후 API-009로 품절 변경. 품절 목록 뷰 필요 (2026-07-06).
상태: 와이어프레임
우선순위: 중
입력 데이터: targetType(MENU/INGREDIENT), targetId, isSoldOut
화면 ID: SCR-011
화면 설명: 관리자가 메뉴 또는 재료의 품절 상태를 변경하는 화면입니다. 품절 변경 후 메뉴·재료 상태 목록을 확인할 수 있습니다.
출력 데이터: 메뉴 목록, 메뉴 기본 재료, 옵션 항목, 재료 역할, 변경된 품절 상태, 영향받는 메뉴/옵션
