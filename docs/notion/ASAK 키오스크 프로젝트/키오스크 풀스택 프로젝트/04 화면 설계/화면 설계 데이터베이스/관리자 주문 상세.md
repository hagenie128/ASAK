# 관리자 주문 상세

개발 완료: No
관련 API: API-007 GET /api/admin/orders 결과 사용, API-008 PATCH /api/admin/orders/{orderId}/status
관련 시나리오: 관리자 주문 상세 확인 / 주문 상태 변경 시나리오
관련 요구사항: LMIS-ORDER-001, LMIS-ORDER-002, LMIS-ORDER-003, LMIS-ORDER-004
관련 테스트: TC-014
구분: 관리자
단계: KSD
담당자: 나연, 하진
도메인: 식당
비고: 별도 주문 상세 API는 Day 10 1차 발표에서 만들지 않고 API-007 목록/상세 데이터 기반으로 표시
상태: 와이어프레임
우선순위: 상
입력 데이터: orderId, 변경할 status
화면 ID: SCR-010
화면 설명: 관리자가 특정 주문의 메뉴, 옵션, 주문유형, 결제상태, 주문상태를 확인하는 화면
출력 데이터: 주문 상세 정보, 변경된 주문 상태

# 화면 목적

관리자가 주문 1건의 상세 정보를 확인하고 상태를 변경합니다.

# 주요 요소

- 주문번호
- 주문유형
- 결제상태
- 주문상태
- 메뉴 및 옵션 목록
- 총 금액
- 상태 변경 버튼

# 상태 변경

RECEIVED → PREPARING → COMPLETED
