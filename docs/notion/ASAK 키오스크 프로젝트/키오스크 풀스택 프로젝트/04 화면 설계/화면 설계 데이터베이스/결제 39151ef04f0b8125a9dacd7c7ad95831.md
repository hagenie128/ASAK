# 결제

Figma 링크: https://www.figma.com/design/iqaoVwFjFE6Zq1WpOVgjeG/kiosk_design?node-id=75-8
개발 완료: No
관련 API: API-006 POST /api/payments
관련 시나리오: 가상 결제 완료 시나리오
관련 요구사항: FWD-PAY-001, KSD-MEMBER-001, KSD-PAY-001
관련 테스트: TC-004, TC-005
구분: 키오스크
단계: KSD
도메인: 식당
비고: 실제 결제 연동이 아닌 가상 결제 승인 흐름. 결제 승인 대기 로딩 상태 필수. 결제 실패·재시도는 SCR-012 팝업/토스트 오버레이 (2026-07-06).
상태: 와이어프레임
우선순위: 상
입력 데이터: orderId, paymentMethod, amount
화면 ID: SCR-007
화면 설명: 주문 금액을 확인하고 실제 PG 연동 없이 가상 결제를 진행하는 화면. 승인 대기 중 로딩 상태 필수.
출력 데이터: paymentId, paymentStatus, orderStatus

<aside>
📌

**2026-07-06 회의 반영** — 결제 승인 **로딩 상태 필수**. 실패·재시도는 SCR-012 팝업/토스트 오버레이.

</aside>

# 화면 목적

주문 금액을 확인하고 실제 PG 연동 없이 **가상 결제**를 진행합니다.

# 주요 요소

- 결제 수단 선택
- 주문 금액 확인
- **결제 승인 로딩** 오버레이 (필수)
- 실패 시 → SCR-012 팝업/토스트

# 이동

- API-006 성공 → 주문 완료(SCR-008)
- 실패 → SCR-012 오버레이 (재결제 또는 SCR-005 복귀)
