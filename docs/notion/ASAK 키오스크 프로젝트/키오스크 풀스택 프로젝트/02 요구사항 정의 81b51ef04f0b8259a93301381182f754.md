# 목적

이 페이지는 키오스크 프로젝트의 요구사항을 데이터베이스로 관리합니다.

요구사항은 문장으로만 정리하면 구현 상태, 우선순위, 관련 화면/API/DB와 연결해서 보기 어렵습니다. 
따라서 요구사항 목록은 인라인 DB로 관리합니다.

## 관리 기준

- 요구사항 ID를 부여합니다.
- `REQ-001`처럼 범위가 불명확한 공통 ID는 사용하지 않고, 
`FWD-MENU-001`, `LMIS-ORDER-001`, `KSD-ORDER-001`처럼 
단계/기능분류/번호가 보이는 형식으로 작성합니다.
- 기능, 비기능, 화면, DB, API, 장치, 관리자 요구사항으로 구분합니다.
- 기능계층은 기본기능(MVP) / 옵션기능 / 추가기능으로 구분합니다.
- 우선순위를 상/중/하로 정합니다.
- 관련 화면, API, DB 테이블, WBS 작업을 함께 적습니다.
- 구현 상태를 계속 업데이트합니다.
- 최종 테스트 시 요구사항이 구현되었는지 확인합니다.
- 기본기능 요구사항은 관련 시나리오, 화면, API 또는 WBS 중 최소 1개 이상 연결되어야 합니다.

## 상태값

예정 / 진행중 / 검토중 / 구현완료 / 테스트완료 / 제외

## 기능계층 기준

- 기본기능: **Week 5 MVP**(8/1 목표) 필수 구현 범위입니다.
- 옵션기능: MVP 흐름에 붙일 수 있지만, 일정에 따라 9주 로드맵 안에서 범위를 조절할 수 있는 기능입니다.
- 추가기능: Week 6 이후 후반 주차 확장 기능입니다.

## 요구사항 작성 예시

- 고객은 키오스크에서 먹고가기/포장을 선택할 수 있어야 한다.
- 고객은 카테고리별 메뉴를 조회하고 옵션을 선택해 주문할 수 있어야 한다.
- 관리자는 주문 목록과 주문 상세를 확인하고 주문 상태를 변경할 수 있어야 한다.
- 관리자는 옵션기능으로 메뉴 또는 재료의 품절 여부를 변경할 수 있어야 한다.
