## 반영 순서

1. 위 중복 ID의 정식 번호를 확정한다.
2. 요구사항 → 시나리오 → 화면 → API → WBS → 테스트 순으로 텍스트 ID·Relation을 갱신한다.
3. 각 DB는 기존 Notion 행을 보존하고, ID가 확정된 행만 업데이트 또는 추가한다.
4. 마지막으로 관련 문서의 일정·DB 테이블명·링크를 재검수한다.
