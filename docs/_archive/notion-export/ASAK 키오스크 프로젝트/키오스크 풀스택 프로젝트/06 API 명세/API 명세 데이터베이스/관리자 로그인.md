# 관리자 로그인

API ID: API-HOLD-001
Error: {"success":false,"status":401,"code":"LOGIN_FAILED","message":"아이디 또는 비밀번호가 올바르지 않습니다.","data":null}
Method: POST
Request: Body: { username, password }
Response: {"success":true,"status":200,"code":"LOGIN_SUCCESS","message":"로그인 성공","data":{"adminId":1,"username":"admin","name":"관리자","role":"ADMIN","token":"sample-token"}}
URL: /api/auth/login
관련 테이블: admin, role (05번 DB 설계 확장 후보 테이블, 22개 정식 테이블 외)
구분: 인증
사용 화면: 관리자 로그인 화면
상태: 보류
설명: Week 5 MVP 범위에서 제외. 관리자 인증과 role_id 기반 권한 관리는 Week 6(SCR-009~011) 이후 확장 기능에서 적용
인증: N
처리 내용: Week 6 MVP에서는 구현하지 않음. 응답 형식은 공통 JSON 래퍼 기준으로만 정리

## 요청 예시

```
POST /api/auth/login
```

```json
{
  "username": "admin",
  "password": "1234"
}
```

## 성공 응답 예시

```json
{
  "success": true,
  "status": 200,
  "code": "LOGIN_SUCCESS",
  "message": "로그인 성공",
  "data": {
    "adminId": 1,
    "username": "admin",
    "name": "관리자",
    "role": "ADMIN",
    "token": "sample-token"
  }
}
```

## 실패 응답 예시

```json
{
  "success": false,
  "status": 401,
  "code": "LOGIN_FAILED",
  "message": "아이디 또는 비밀번호가 올바르지 않습니다.",
  "data": null
}
```

## 처리 기준

- Week 6(SCR-009~011) MVP 범위에서는 구현하지 않는 보류 API다.
- 이후 관리자 인증과 `role_id` 기반 권한 관리가 필요할 때 확장한다.
- 보류 API라도 응답 형식은 공통 JSON 래퍼 기준을 따른다.
