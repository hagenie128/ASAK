# 화면 목적

담은 메뉴·옵션·수량·총액을 확인·수정합니다. **결제 전 최종 확인은 컨펌 팝업**으로 처리하며, 컨펌 시 API-005로 주문을 생성합니다.

# 주요 요소

- 주문 유형: 먹고가기 / 포장 (`orderType`)
- 메뉴 목록 (썸네일·옵션 요약) — 수량 ± / 삭제
- 예상 총액
- 하단 **추가 제안 상품** 섹션
- **결제하기** CTA → 컨펌 팝업 → API-005
- 알레르기·칼로리는 **SCR-004**에서 표시 (본 화면 아님)

# 이동

- ← 뒤로 → 메뉴 상세(SCR-004)
- **결제하기** + 컨펌 확인 → API-005 성공 → 결제(SCR-007)
- 주문번호는 결제 완료(SCR-008) 후 표시
