# ASAK 기능별 구현 순서표

> 이 문서는 프론트엔드와 백엔드가 함께 구현해야 할 기능을 화면/로직 기준으로 순서대로 정리한 가이드입니다.

---

## 1. 구현 우선순위 기준

프로젝트는 아래 순서로 구현하는 것이 가장 자연스럽습니다.

1. 화면이 먼저 보이게 만든다.
2. 화면에서 필요한 데이터를 연결한다.
3. 사용자의 선택 상태를 관리한다.
4. 주문 생성까지 연결한다.
5. 관리자 기능으로 확장한다.

즉, “보이는 화면”부터 만들고, 그다음 데이터와 로직을 붙이는 방식입니다.

---

## 2. 구현 순서 1: 기본 화면 구성

### 목표

- 앱이 실행되면 화면이 보이도록 한다.
- 메인 레이아웃과 페이지 전환이 가능하도록 한다.

### 프론트엔드 작업

- 앱 진입 페이지 구성
- 공통 레이아웃 구성
- 라우팅 설정
- 기본 스타일 적용

### 백엔드 작업

- 서버가 정상 실행되는지 확인
- 기본 health API 확인

### 완료 기준

- 앱 실행 시 화면이 정상 표시된다.
- 페이지 이동이 가능하다.

---

## 3. 구현 순서 2: 메뉴 조회 흐름

### 목표

- 사용자가 메뉴를 볼 수 있게 한다.

### 프론트엔드 작업

- 카테고리 목록 화면 구성
- 메뉴 목록 화면 구성
- 메뉴 카드 UI 구성
- API 호출해서 메뉴 데이터 표시
- 로딩/에러/빈 상태 처리

### 백엔드 작업

- 카테고리 목록 API 구현
- 메뉴 목록 API 구현
- 메뉴 상세 API 구현

### 완료 기준

- 메뉴 목록이 화면에 보인다.
- 카테고리 선택 시 해당 메뉴가 바뀐다.

---

## 4. 구현 순서 3: 메뉴 상세 / 옵션 선택 흐름

### 목표

- 사용자가 메뉴를 선택하고 옵션을 고를 수 있게 한다.

### 프론트엔드 작업

- 메뉴 상세 페이지 구성
- 옵션 그룹/옵션 아이템 UI 구성
- 선택 상태 관리
- 가격 계산 로직 반영
- 선택값이 화면에 즉시 반영되도록 처리

### 백엔드 작업

- 메뉴 상세 조회 API에 옵션 정보 포함
- 옵션 그룹/옵션 아이템 데이터 제공
- 가격 계산 기준 확인

### 완료 기준

- 메뉴 상세에서 옵션 선택이 가능하다.
- 선택한 옵션에 따라 가격이 바뀐다.

---

## 5. 구현 순서 4: 장바구니 흐름

### 목표

- 선택한 메뉴와 옵션을 장바구니에 담고 관리할 수 있게 한다.

### 프론트엔드 작업

- 장바구니 상태 관리
- 메뉴 추가/수량 변경/삭제 기능
- 총액 계산
- 장바구니 화면 구성
- 장바구니 비우기 처리

### 백엔드 작업

- 장바구니 검증 API 설계
- 주문 전 확인용 데이터 처리

### 완료 기준

- 메뉴를 담으면 장바구니에 보인다.
- 수량과 금액이 정상 반영된다.

---

## 6. 구현 순서 5: 주문 생성 흐름

### 목표

- 장바구니를 주문으로 생성할 수 있게 한다.

### 프론트엔드 작업

- 주문 확인 화면 구성
- 주문 요청 데이터 구성
- 결제 전 확인 화면 구현
- API 요청 후 완료 화면 전환

### 백엔드 작업

- 주문 생성 API 구현
- 주문 엔티티 생성
- 주문 항목 저장
- 주문 번호/주문 상태 생성

### 완료 기준

- 주문 생성 요청이 성공하면 주문 결과가 반환된다.
- 프론트에서 완료 화면으로 이동한다.

---

## 7. 구현 순서 6: 결제 흐름

### 목표

- 주문에 대한 결제 상태를 처리한다.

### 프론트엔드 작업

- 결제 화면 구성
- 결제 요청/응답 처리
- 결제 성공/실패 화면 구현

### 백엔드 작업

- 결제 승인/실패 처리 API 구현
- 결제 상태 저장
- 주문 상태 변경

### 완료 기준

- 결제 성공 시 주문 상태가 반영된다.
- 실패 시 사용자에게 오류 메시지가 보인다.

---

## 8. 구현 순서 7: 관리자 기능

### 목표

- 관리자 화면에서 주문과 메뉴 상태를 관리할 수 있게 한다.

### 프론트엔드 작업

- 관리자 로그인/메뉴 관리/주문 관리 화면 구성
- 주문 목록 및 상태 변경 UI
- 품절 상태 변경 UI

### 백엔드 작업

- 관리자 주문 목록 API 구현
- 주문 상태 변경 API 구현
- 품절 상태 변경 API 구현
- 메뉴 관리 API 구현

### 완료 기준

- 관리자 화면에서 주문 상태를 확인하고 변경할 수 있다.

---

## 9. 기능별 파일 구현 포인트

### 메뉴 기능

- 프론트: `pages/MenuListPage`, `features/menu-selection`
- 백엔드: `domain/menu/controller`, `domain/menu/service`, `domain/menu/repository`

### 장바구니 기능

- 프론트: `features/cart`
- 백엔드: `domain/order` 또는 `domain/cart` 관련 서비스

### 주문 기능

- 프론트: `pages/CheckoutPage`, `pages/CompletePage`
- 백엔드: `domain/order`

### 결제 기능

- 프론트: `features/payment`
- 백엔드: `domain/payment`

### 관리자 기능

- 프론트: `pages/AdminPage`, `features/admin-order`
- 백엔드: `domain/admin`

---

## 10. 구현할 때 꼭 기억할 순서

개발은 항상 아래 순서로 접근하면 좋습니다.

```text
화면 구성
→ 데이터 연결
→ 상태 관리
→ API 연동
→ 비즈니스 로직 처리
→ 결과 확인
```

즉, “보이는 화면”을 먼저 만들고, 그다음 데이터와 로직을 붙이는 방식이 가장 실수 없이 진행됩니다.

---

## 11. 추천 작업 템플릿

새 기능을 시작할 때는 아래 방식으로 진행하면 좋습니다.

1. 화면 먼저 구성한다.
2. 필요한 데이터/상태를 정리한다.
3. API 요청 위치를 정한다.
4. 백엔드와 계약을 맞춘다.
5. 서버 응답을 화면에 반영한다.
6. 오류/빈 상태를 처리한다.

---

## 12. 한 문장 요약

ASAK은 “메뉴 조회 → 옵션 선택 → 장바구니 → 주문 생성 → 결제 → 관리자 관리” 순서로 구현하는 프로젝트이므로, 기능을 구현할 때도 이 순서로 진행하는 것이 가장 자연스럽습니다.
