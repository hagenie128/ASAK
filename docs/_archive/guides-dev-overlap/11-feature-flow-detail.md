# ASAK 기능별 상세 흐름 가이드

> 이 문서는 메뉴 선택부터 주문 완료까지의 흐름을 프론트엔드와 백엔드가 각각 어떤 책임을 갖는지 중심으로 정리한 가이드입니다.

---

## 1. 전체 흐름 요약

ASAK의 핵심 기능은 아래 순서로 진행됩니다.

```text
메뉴 조회
→ 메뉴 상세 확인
→ 옵션 선택
→ 장바구니 담기
→ 주문 확인
→ 결제 요청
→ 주문 완료
```

각 단계는 프론트엔드와 백엔드가 분담해서 처리합니다.

---

## 2. 1단계: 메뉴 조회 흐름

### 사용자 관점

- 홈 화면에서 카테고리를 확인한다.
- 원하는 카테고리를 선택한다.
- 메뉴 목록을 본다.

### 프론트엔드 처리

- 카테고리 선택 상태를 변경한다.
- 메뉴 목록 API를 호출한다.
- 응답을 받아 카드 형태로 렌더링한다.
- 로딩/에러 상태를 표시한다.

### 백엔드 처리

- 카테고리 목록 API를 제공한다.
- 메뉴 목록 API를 제공한다.
- 메뉴에 필요한 기본 정보(이름, 가격, 이미지, 설명)를 반환한다.

### 핵심 포인트

- 프론트는 “보여주는 역할”
- 백엔드는 “데이터 제공 역할”

---

## 3. 2단계: 메뉴 상세 / 옵션 선택 흐름

### 사용자 관점

- 메뉴를 누른다.
- 메뉴 설명과 옵션을 확인한다.
- 원하는 옵션을 선택한다.

### 프론트엔드 처리

- 메뉴 상세 화면을 표시한다.
- 옵션 그룹별로 선택 UI를 렌더링한다.
- 사용자의 선택을 상태로 저장한다.
- 옵션 선택에 따라 총 가격을 즉시 계산한다.
- 선택 완료 후 장바구니 담기 버튼을 활성화한다.

### 백엔드 처리

- 메뉴 상세 API에서 옵션 정보도 같이 제공한다.
- 옵션 이름, 가격, 필수 여부, 선택 가능 수량 등을 반환한다.
- 필요하면 가격 계산 기준을 함께 내려준다.

### 핵심 포인트

- 프론트는 “선택 상태를 관리”
- 백엔드는 “옵션 데이터를 제공”

---

## 4. 3단계: 장바구니 담기 흐름

### 사용자 관점

- 옵션을 선택한 뒤 장바구니에 담는다.
- 장바구니에서 수량을 조정한다.

### 프론트엔드 처리

- 선택한 메뉴와 옵션을 장바구니 상태에 추가한다.
- 수량 변경 시 총액을 재계산한다.
- 장바구니 화면에서 항목을 표시한다.
- 삭제·수정 기능을 연결한다.

### 백엔드 처리

- 주문 전 장바구니 검증용 로직을 제공할 수 있다.
- 필요 시 장바구니 항목의 유효성 확인을 처리한다.

### 핵심 포인트

- 장바구니는 프론트의 상태 관리가 중심이다.
- 백엔드는 장바구니를 저장소로 다루거나 검증 로직을 제공한다.

---

## 5. 4단계: 주문 확인 흐름

### 사용자 관점

- 장바구니를 확인한다.
- 주문 정보를 최종 확인한다.
- 주문하기 버튼을 누른다.

### 프론트엔드 처리

- 장바구니 내용을 주문 폼으로 구성한다.
- 주문 타입(먹고가기/포장), 요청 사항, 선택 항목을 정리한다.
- 서버에 주문 생성 요청을 보낸다.

### 백엔드 처리

- 주문 요청 DTO를 검증한다.
- 주문 생성 로직을 실행한다.
- 주문 엔티티와 주문 아이템을 저장한다.
- 주문 번호와 상태를 응답으로 반환한다.

### 핵심 포인트

- 프론트는 “주문 데이터를 구성”
- 백엔드는 “주문을 생성”

---

## 6. 5단계: 결제 흐름

### 사용자 관점

- 결제 화면에서 결제를 진행한다.
- 성공/실패 결과를 확인한다.

### 프론트엔드 처리

- 결제 요청 화면을 표시한다.
- 결제 결과를 받아 화면에 반영한다.
- 성공 시 완료 화면으로 이동한다.
- 실패 시 오류 메시지를 보여준다.

### 백엔드 처리

- 결제 승인/실패 처리를 수행한다.
- 결제 상태와 주문 상태를 갱신한다.
- 결과를 응답으로 반환한다.

### 핵심 포인트

- 결제는 백엔드 비즈니스 로직이 핵심이다.
- 프론트는 결과를 사용자에게 전달하는 역할이다.

---

## 7. 6단계: 주문 완료 흐름

### 사용자 관점

- 주문 완료 화면을 확인한다.
- 영수증/주문 번호를 확인한다.

### 프론트엔드 처리

- 주문 완료 화면을 표시한다.
- 서버 응답으로 받은 주문 번호/상태를 보여준다.
- 다시 주문하거나 홈으로 가는 동작을 제공한다.

### 백엔드 처리

- 완료된 주문 정보를 확인 가능한 응답으로 반환한다.

### 핵심 포인트

- 사용자에게 “무엇이 완료됐는지”를 명확히 보여주는 단계다.

---

## 8. 기능별 책임 분담 요약

| 단계      | 프론트엔드          | 백엔드             |
| --------- | ------------------- | ------------------ |
| 메뉴 조회 | 화면 표시, API 호출 | 메뉴/카테고리 조회 |
| 메뉴 상세 | 옵션 UI, 상태 관리  | 옵션 데이터 제공   |
| 장바구니  | 상태 관리, 계산     | 검증/저장 로직     |
| 주문 확인 | 주문 데이터 구성    | 주문 생성          |
| 결제      | 결과 표시           | 결제 처리          |
| 주문 완료 | 완료 화면 표시      | 완료 정보 응답     |

---

## 9. 구현할 때 가장 먼저 알아야 할 로직

### 프론트엔드에서 꼭 이해해야 할 것

- 사용자가 어떤 액션을 했는지
- 그 액션이 화면의 어떤 상태를 바꾸는지
- 상태 변경 후 화면이 다시 그려지는지

### 백엔드에서 꼭 이해해야 할 것

- 어떤 요청이 들어오는지
- 어떤 데이터를 저장/조회해야 하는지
- 어떤 비즈니스 규칙이 적용되는지

---

## 10. 개발 순서로 보면 좋은 방식

```text
1. 메뉴 조회 화면
2. 메뉴 상세/옵션 선택
3. 장바구니 상태 관리
4. 주문 확인 화면
5. 주문 생성 API 연결
6. 결제 흐름 연결
7. 완료 화면 구성
```

이 순서로 진행하면 기능이 자연스럽게 연결됩니다.

---

## 11. 한 문장 요약

ASAK은 “사용자가 메뉴를 보고 선택하고, 장바구니에 담고, 주문을 생성하고, 결제까지 이어지는 흐름”으로 구현되므로, 개발은 화면 흐름 기준으로 연결해서 진행하는 것이 가장 효과적입니다.
