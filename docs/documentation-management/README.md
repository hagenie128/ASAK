## 앞으로 문서를 추가할 때

- 새 문서보다 **현재 정본 한 곳 갱신**을 먼저 선택한다.
- 작업 문서는 `목적 → 지금 할 일 → 완료 기준 → 필요한 링크` 순서의 요점형으로 쓴다.
- 날짜별 회의·AI 대화·감사·수정 계획은 활성 폴더가 아니라 `_archive/YYYY-MM-DD-주제/`에 넣고 README 한 줄로만 안내한다.
- 구현 결과, 문제 해결 과정, 검증 결과, 화면·PR 링크는 `worklog/entries/`에 남긴다. 이것이 포트폴리오 근거다.
