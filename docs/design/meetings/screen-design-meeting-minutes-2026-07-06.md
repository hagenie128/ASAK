## 결정 사항

### 디자인 시스템

| 항목 | 결정 |
|------|------|
| **최종 DS** | **DS-02 Modern Minimal** (charcoal + electric lime) — [candidate-B](../kiosk-design-system-candidate-B.md) |
| **박유진 1순위** | **DS-04** (DS-01 vs DS-04 재투표, 안건 7) |
| **미채택 참고안** | DS-02 기반 하이브리드 아이디어를 논의했으나 채택하지 않음. 후보 정리 후 문서는 보관하지 않음. |
| **고객/관리자 DS** | **일단 통합**, 분리 톤은 추후 검토 |

### 화면 · UX

| 주제 | 결정 |
|------|------|
| **화면 총수** | 18개 축소보다 **유지·증가** 쪽 (신규·병합 조합으로 SCR DB 재정리 필요) |
| **홈·포장/매장** | **SCR-001·002 병합** — 홈에 매장/포장 선택 표시 |
| **장바구니 ↔ 주문확인** | **기능 병합** — 최종 확인은 **팝업(컨펌)** 만; 주문번호는 **결제 완료 후**에만 표시 |
| **추가 제안 상품** | 장바구니 **하단** 배치 (추천 우선 모드일 때는 생략) |
| **알레르기·칼로리** | 장바구니가 아닌 **메뉴 선택(옵션) 화면**에 표시 |
| **추천·빠른 주문** | 메뉴 선택 페이지에서 고려, **MVP 후순위** (Phase 2) |
| **결제 UX** | 승인 대기 **로딩 화면 필수**; 실패·재시도는 **팝업·토스트** (박유진 의견 채택) |
| **멤버십·영수증** (SCR-020, 021) | **나중에 고려** |
| **관리자 분석** (SCR-019) | 매출 평균, **시간대별·판매 제품 통계** |
| **관리자 주문** | **기존 기능** 유지 |
| **품절 처리** (SCR-011) | 품절 변경 후 **메뉴 상태 목록** — **필요** |

### 합의 통과 (사전의견과 동일)

- 결제 대기 시 로딩 화면
- 결제 실패·재시도 팝업/토스트
- 관리자 화면 가독성·정보 밀도 우선
