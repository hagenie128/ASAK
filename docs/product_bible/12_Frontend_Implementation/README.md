# ASAK Product Bible Pack 12 — Frontend Implementation Bible

> Status: Current Draft  
> Repositories: ASAK-Kiosk · ASAK_Admin  
> Stack: React · JavaScript · Zustand · Axios · Vite

## 목적

기존 팀원이 작성한 고객용 Frontend와 함께 만든 Figma를 최대한 보존하면서 실제 API를 연결한다.

## 핵심 원칙

1. 기존 코드를 먼저 읽고 재사용한다.
2. 새 문서 구조에 맞추기 위해 기존 코드를 갈아엎지 않는다.
3. 이름이 달라도 역할이 같으면 기존 컴포넌트를 확장한다.
4. 화면 전체 재작성보다 누락 상태·props·API 연결을 보완한다.
5. Kiosk와 Admin을 분리하되 공통 규칙은 동일하게 사용한다.
6. Mock Data에서 실제 API로 교체할 수 있는 구조를 유지한다.
