### Figma–React 화면·데이터 계약 정리 — `frontend`

- **작업 목적:** Figma에서 설계한 키오스크·관리자 화면이 React 구현 시 어떤 페이지, 데이터, 상태와 연결되는지 명확히 한다.
- **직접 구현:** `requirements-screen-map.md`를 기준으로 키오스크 `k001~k007`, 관리자 `A-001~A-007`을 React 페이지 후보와 매핑하고, 화면별 데이터 요구사항 및 로딩·빈 목록·오류·품절·타임아웃 상태를 정리했다. `cartStore.js`, `orderStore.js`의 자리표시자 구조와 장바구니·결제 데이터 계약도 확인했다.
- **AI 도움:** Figma MCP로 파일 구조와 화면·컴포넌트 후보를 점검하고, 매핑 초안 검토에 보조적으로 활용했다.
- **이슈:** 장바구니·주문 상태 저장소와 디자인 토큰은 실제 구현이 필요한 자리표시자 상태다.
- **검증:** 요구사항 맵, Figma 프레임 목록, 프론트엔드 페이지·스토어·컴포넌트 후보 경로를 대조했다.
- **포트폴리오 요약:** 디자인 산출물을 화면·상태·데이터 계약으로 전환해 Figma와 React 구현 사이의 단절을 줄였다.

### 컴포넌트 코드 연동 준비 — `design-system`

- **작업 목적:** 재사용 가능한 Figma 컴포넌트를 React props와 Code Connect 후보 구조에 맞춘다.
- **직접 구현:** Button, MenuCard, CategoryTab, CartItem, OptionCategory, PaymentMethodCard, Badge, DataTable의 Variant/Property와 React 컴포넌트 후보를 연결했다. `CartItem_v2`의 상태·옵션·수량·합계 구조 및 Figma 변수에서 `tokens.css`로 옮길 명명 기준을 정리했다.
- **AI 도움:** Figma MCP를 통한 구조 점검에 보조적으로 활용했고, 적용 우선순위와 구현 전 정리 항목은 직접 검토했다.
- **이슈:** 자동 생성 레이어·Variant 이름, 중복 MenuCard 명칭, 접근성·예외 상태가 구현 전 정리 항목으로 남아 있다.
- **검증:** [`Figma MCP review archive`](../../../docs/archive/design-audits/ASAK_FIGMA_MCP_REVIEW_2026-07-14.md)에 화면 매핑, 컴포넌트 Property, 토큰, 체크리스트를 기록했다.
- **포트폴리오 요약:** Figma Variant를 React props와 디자인 토큰으로 연결하는 개발 핸드오프 기준을 설계했다.
