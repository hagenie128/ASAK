  - `k001~k007`과 관리자 `A-001~A-007`을 React 페이지 후보(`HomePage`, `MenuListPage`, `CartPage`, `PaymentPage`, `OrderListPage` 등)로 매핑하고, 화면별 로딩·빈 목록·오류·품절·타임아웃 상태를 정의했다.

- **AI 도움:** Figma MCP를 통한 구조 점검과 화면/컴포넌트 매핑 초안에 보조적으로 활용했고, 구현 우선순위와 필요한 상태는 직접 검토했다.
- **검증:** 화면 요구사항 맵, Figma 프레임 목록, 프론트엔드 디렉터리의 페이지·스토어·컴포넌트 후보 경로를 대조했다.
- **포트폴리오 요약:** 디자인 산출물을 화면·상태·데이터 계약으로 전환해 Figma와 React 구현 사이의 단절을 줄였다.

### 컴포넌트 코드 연동 준비 — `design-system`

- **작업 목적:** 재사용 가능한 Figma 컴포넌트를 React props와 Code Connect 후보 구조에 맞춘다.
- **직접 구현:**
  - Button, MenuCard, CategoryTab, CartItem, OptionCategory, PaymentMethodCard, Badge, DataTable의 Figma Variant/Property와 React 컴포넌트 후보를 연결했다.
  - `CartItem_v2`의 Default/Disabled, 옵션, 수량, 상품별 합계를 코드 props·상태로 옮길 수 있도록 구조를 점검했다.
  - Figma Palette·Semantic 변수, 텍스트/효과 스타일을 `tokens.css`의 CSS 변수로 옮기기 위한 명명·우선순위 기준을 문서화했다.
  - 자동 생성된 레이어/Variant 이름, 중복 MenuCard 명칭, 누락된 접근성·예외 상태를 구현 전 정리 항목으로 남겼다.
- **검증:** [`docs/design/ASAK_FIGMA_MCP_REVIEW_2026-07-14.md`](../../../docs/design/ASAK_FIGMA_MCP_REVIEW_2026-07-14.md)에 화면 매핑, 컴포넌트 Property, 토큰, 구현 전 체크리스트를 기록했다.
- **포트폴리오 요약:** Figma 컴포넌트의 Variant를 React props와 디자인 토큰으로 연결하는 개발 핸드오프 기준을 설계했다.
