### Figma Final 컴포넌트·Cart 상태 정책 검수 — SCR-003·004·005·007·009·010·016·019 · `design` / `docs`

- **작업 목적:** C 버전의 구현 정본 역할을 유지하면서 Kiosk/Admin 컴포넌트 구조, Variant, 화면 상태 및 Cart 품절·수량 정책을 실제 구현 가능한 기준으로 정리한다.
- **직접 구현:** Kiosk/Admin Components Implementation Final 복제본을 기준으로 중복 컴포넌트, 오타 Variant, Legacy·Final 혼용, Shared 승격 후보를 판단했다. Cart의 수량 1일 때 minus 비활성, 동일 메뉴 9개·전체 30개 제한, 품절 후 자동 삭제 금지·수정/삭제 유도·결제 차단 정책을 확정했다.
- **AI 도움:** ChatGPT·Figma MCP/Figma Agent에게 원본을 보존하는 복제·노드/Variant 감사·정확한 수정 지시·결과 보고 형식을 요청했다. Node ID, Component Set, 인스턴스 수, Detach 여부는 AI 보고를 근거로 재검토했다.
- **이슈:** `PaymentMethodCard` 중복 Variant, `OrderDetailRow` 속성명 불일치, `StatusBadge`/`Badge` 완전 중복, `DatePicker`/`DateRangePicker`의 Frame 상태, 품절 Cart의 CTA 활성 표기가 발견됨 → Final Master 정리와 화면별 결제 차단 기준을 후속 작업으로 분리했다.
- **검증:** Figma Agent 보고의 Component Set·Variant·인스턴스·수동 opacity·Detach 수를 대조하고, Cart 품절 3개 상태에서 결제 CTA가 disabled여야 한다는 정책을 확인했다.
- **포트폴리오 요약:** Figma 화면을 예쁘게 다듬는 수준을 넘어 컴포넌트 정본, 화면 상태, Cart 정책과 개발 구현 조건을 연결해 검수 기준을 만들었다.
