# 상세 워크로그 (entries)

기능·이슈·PR 단위의 **상세** 작업 기록을 둡니다. 일일 `daily/`와 **같은 섹션 이름**을 쓰되, 여기서만 12섹션 전체를 채웁니다.

## daily와의 관계

| | `daily/YYYY-MM-DD.md` | `entries/` (이 폴더) |
|---|---|---|
| 단위 | 하루 | 기능·이슈 하나 |
| daily 역할 | **오늘 요약** 표 1줄 + **오늘 작업** 미니 카드 + 이 파일 링크 | — |
| entries 역할 | — | [03-work-log-template.md](../../docs/guides/03-work-log-template.md) 12섹션 전체 |

자세한 흐름: [03-work-log-template § 일일 워크로그와의 관계](../../docs/guides/03-work-log-template.md) · [guide-team-daily § 예시](../guide-team-daily.md#daily--entries-예시)

## 작성 규칙

1. [`docs/guides/03-work-log-template.md`](../../docs/guides/03-work-log-template.md)를 복사해 시작
2. 파일명: `YYYY-MM-DD-짧은-설명.md` (예: `2026-07-02-scr-003-menu-option-ui.md`)
3. 같은 날짜 [`daily/`](../daily/) — **오늘 요약** 표 한 줄 + **오늘 작업** 미니 카드 + 이 파일 링크
