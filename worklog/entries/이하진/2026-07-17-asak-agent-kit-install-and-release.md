## 2. 작업 목적

- 팀원이 사용하는 AI 도구가 달라도 ASAK 프로젝트의 공통 규칙, 안전 원칙, 단축 명령을 쉽고 같은 방식으로 설치하게 한다.
- 설치 과정을 비개발자도 따라 할 수 있게 단순화하고, Windows 배치 파일의 한글 깨짐 문제를 제거한다.

## 3. 직접 구현 영역

- `sources/skills/`에 가격·수량·주문·React·백엔드·DB·API·Figma·버그·테스트·설명·배포·발표·오늘·이어서 등 ASAK 작업용 공통 스킬을 구성했다.
- `packages/`에 Codex, Claude Code, Cursor, GitHub Copilot, Antigravity, ChatGPT 환경별 설치 패키지와 지침 변환본을 배치했다.
- Windows 설치·검증 도구(`install-asak`, `verify-asak`)와 README·수동 설치 안내를 구성했다.
- v2 설치 흐름의 복잡함과 `.bat` 한글 깨짐을 확인한 뒤 v2.1에서 AI 선택, 폴더 선택, PowerShell 한글 안내, 단일 안내 파일 중심으로 사용성을 개선했고, 사용자 확인 기준 Git에 배포했다.

## 4. 구현 로직 / 적용한 방식

- 공통 규칙은 원본 스킬로 관리하고, 각 AI 도구가 읽는 형식(예: AGENTS.md, CLAUDE.md, GEMINI.md, 플러그인 스킬)에 맞춰 패키지를 나눴다.
- 설치기는 사용자가 프로젝트 내부의 `src` 또는 `package.json`이 있는 폴더를 선택하도록 해 엉뚱한 상위 폴더에 설치하는 위험을 줄였다.
- 배치 파일은 영문 중심으로 실행하고, 한글 안내는 PowerShell에서 보여 주도록 분리해 인코딩 문제를 피했다.
- 설치 후 AI 도구를 다시 열고 한 단어 명령(예: `가격`, `버그`, `피그마`)으로 스킬을 호출하는 흐름을 안내했다.

## 5. AI 도움 영역

- 사용한 AI 도구: Codex, ChatGPT
- 어떤 질문/요청을 했는지: 팀용 AI 스킬의 폴더 구조, 도구별 설치 방식, 안전 규칙, 짧은 한국어 명령 체계, Windows 설치기 UX를 설계·수정하도록 요청했다.
- AI가 도움 준 내용: 스킬 분류, 도구별 지침 파일 형태, 설치/검증 스크립트 초안, README와 쉬운 설치 안내 초안을 제안했다.
- 그대로 사용한 부분: 도구별 패키지 분리 구조와 설치 순서의 기본 틀, 공통 스킬 이름 체계.
- 수정해서 사용한 부분: 번호 입력·경로 붙여넣기 방식은 사용성이 낮아 AI 선택창·폴더 선택으로 바꾸고, 한글 파일명 호출 때문에 생긴 `.bat` 깨짐은 영문 실행부와 PowerShell 안내로 분리했다.
- AI 사용 경계: AI가 제안한 설치 구조를 그대로 배포하지 않고, 실제 사용 중 발견한 인코딩·설명 난이도 문제를 직접 수정한 뒤 Git 배포 범위를 결정했다.

## 6. 발생 이슈

- 이슈 1:
  - 증상: 초기 설치 안내가 길고, 번호·경로를 직접 입력해야 하며 `.bat`에서 한글이 깨져 팀원이 설치를 어려워할 수 있었다.
  - 원인: Windows 콘솔 인코딩과 한글 파일명 호출, 기술 용어 중심의 설명, 수동 입력 중심 설치 흐름.
  - 해결: `.bat` 내부 영문 사용, PowerShell 한글 안내, AI 선택창·폴더 선택, `0_이것만_보세요` 수준의 단일 안내 흐름으로 개선했다.

## 7. 디버깅 기록

- 확인한 로그/에러 메시지: `.bat`에서 한글 파일명을 직접 불러오는 경우 문자가 깨지는 사용성 문제를 확인했다.
- 의심했던 지점: 도구별 설정 폴더가 서로 충돌하는지, 프로젝트 루트를 잘못 선택할 가능성이 있는지, 기존 v2와 새 v2.1을 함께 설치해야 하는지.
- 실제 원인: 사용자가 도구별 설치 구조와 Windows 인코딩 차이를 직접 이해해야 하는 흐름이었다.
- 다시 같은 문제가 생기면 먼저 볼 파일/명령어: `asak-agent-kit/README.md`, `docs/manual-install.md`, `installers/install-asak.cmd`, `installers/install-asak.ps1`, `installers/verify-asak.ps1`.
