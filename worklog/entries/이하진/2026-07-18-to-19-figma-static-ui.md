## 2. 작업 목적

- 0718 Figma 정본을 Kiosk·Admin 화면에 반영하고, 화면별 정적 상태와 공통 스타일을 실제 코드 구조에 맞춘다.

## 3. 직접 구현 영역

- Kiosk Home·메뉴·상세·결제·완료·접근성 화면과 상태 미리보기, 헤더 자산, 공통 UI 규칙을 Figma 기준으로 반영했다.
- Admin 주문 현황·품절·메뉴 관리·매출·대시보드·결제수단·로그인/메뉴 편집 화면 및 상단 헤더를 Figma 기준으로 반영했다.

## 4. 구현 로직 / 적용한 방식

- Kiosk는 Home → Menu → Detail → Cart의 기존 mock/store 동작을 보존하고 Payment 이후 화면은 UI shell로 분리했다.
- Admin은 정적 화면과 mock 데이터 준비 상태를 분리해, 페이지 직접 데이터 import 대신 repository/hook 연결을 다음 단계로 남겼다.

## 5. AI 도움 영역

- Codex가 Git 커밋, Figma 핸드오프 문서, 구현 계획을 대조해 작업 범위와 미연결 상태를 워크로그에 정리했다.

## 6. 발생 이슈

- 이슈 1:
  - 증상: 정적 화면이 Figma와 맞아도 결제·주문·품절·매출 데이터는 실제 동작하지 않았다.
  - 원인: UI 이식 단계와 mock 상태 연결 단계가 분리돼 있었다.
  - 해결: 화면 이식 완료를 기능 완료로 표시하지 않고, 후속 WBS와 GitHub Issue로 연결했다.

- 다시 같은 문제가 생기면 먼저 볼 파일: Kiosk `docs/figma-ui-handoff.md`, Admin `docs/ui-implementation-map.md`.
